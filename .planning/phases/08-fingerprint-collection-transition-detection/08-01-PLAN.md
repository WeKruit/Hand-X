<objective>
Add lightweight DOM fingerprinting to browser_use so SPA page transitions (same URL, different content) are detected and trigger the existing PAGE UPDATE mechanism.

Purpose: On Workday and similar SPA ATS platforms, the URL stays the same across sections. Without fingerprinting, `_page_identity()` returns the same value, PAGE UPDATE never fires, stale context accumulates, and the agent loses track of which page it is on.

Output: Three modified browser_use files (views.py, agent/service.py, message_manager/service.py) plus a unit test file.

```python
def _page_identity(self, browser_state_summary: BrowserStateSummary) -> str:
    url = (browser_state_summary.url or '').strip()
    title = (browser_state_summary.title or '').strip()
    elem_count = 0
    try:
        if browser_state_summary.dom_state and browser_state_summary.dom_state.selector_map:
            elem_count = len(browser_state_summary.dom_state.selector_map)
    except Exception:
        pass
    elem_bucket = (elem_count // 5) * 5
    return f'{title}\n{url}\n{elem_bucket}'
```

    Add a fingerprint JS snippet as a module-level constant (near the top of service.py, outside the class):

    ```python
    _PAGE_FINGERPRINT_JS = r"""() => {
        function texts(sel, n) {
            return Array.from(document.querySelectorAll(sel))
                .map(el => (el.textContent || '').replace(/\s+/g, ' ').trim())
                .filter(Boolean).slice(0, n);
        }
        return JSON.stringify({
            h: texts('h1, h2, h3, [role="heading"]', 6),
            b: texts('button, [role="button"], input[type=\"submit\"]', 8),
            f: document.querySelectorAll('form').length
        });
    }"""
    ```

    This is a simplified version of the reference `_PAGE_FINGERPRINT_JS` in domhand_click_button.py:419-447. Key differences: no `window.__ff` (not needed — headings are in main document per CONTEXT.md decision), shorter property names (h/b/f — this is for hashing, not display), no errorCount/passwordInputs (not relevant for page identity).

    Then in `_prepare_context`, after `browser_state_summary` is obtained and before `prepare_step_state`:

    ```python
    # Collect lightweight DOM fingerprint for SPA transition detection
    try:
        page = await self.browser_session.get_current_page()
        if page:
            raw = await page.evaluate(_PAGE_FINGERPRINT_JS)
            if raw:
                import hashlib
                browser_state_summary.page_fingerprint = hashlib.md5(raw.encode()).hexdigest()[:12]
    except Exception:
        pass  # page_fingerprint stays '' — identity falls back to URL+title+elem_count
    ```

    Use md5 truncated to 12 hex chars — this is not security-critical, just a compact fingerprint. 12 chars = 48 bits = collision-free for the ~10s of pages in one session. The hashlib import can be at module top instead of inline — use judgment.

    This satisfies FPRINT-01: browser-use collects a lightweight page fingerprint per step as part of browser state extraction.
