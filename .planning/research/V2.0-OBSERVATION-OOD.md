## 1. Context: How It Works Today

The current observation is a single function: `extract_form_fields()` in `field_extractor.py`.

It works like this:

```
page.evaluate(BIG_JS_FUNCTION)
        │
        ▼
  JS walks the DOM tree
  JS resolves field types        ──── all in one pass, inside the browser
  JS resolves labels
  JS reads values & options
  JS tags elements with data-ff-id
        │
        ▼
  Returns list[dict] to Python
        │
        ▼
  Python groups radios/checkboxes
  Python validates via Pydantic
        │
        ▼
  list[FormField]  →  given to domhand_fill  →  given to LLM
```

**The problem:** The JS blob does extraction AND interpretation simultaneously.
You can't test the interpretation logic without a browser. You can't swap out the
data source. You can't reuse interpretation for a different extraction strategy.
And the interpretation is full of platform-specific hardcoded checks.

**Key file locations (current system):**

| File | Role |
|------|------|
| `ghosthands/dom/field_extractor.py` | JS injection + extraction + grouping (866 lines) |
| `ghosthands/dom/views.py` | `FormField`, `ExtractionResult` models |
| `ghosthands/dom/shadow_helpers.py` | `__ff` helper injection, `INTERACTIVE_SELECTOR` |
| `ghosthands/dom/fill_executor.py` | Consumes `FormField` for fill operations |
| `ghosthands/actions/domhand_fill.py` | Main fill orchestrator, consumes `list[FormField]` |
| `browser_use/dom/service.py` | Already fetches DOM + a11y tree + snapshot (unused by us) |
| `browser_use/dom/views.py` | `EnhancedDOMTreeNode`, `EnhancedAXNode` (unused by us) |

**The contract between observation and action is `list[FormField]`.**
The new observer must produce `list[FormField]` that is indistinguishable

## 3. Architecture Overview

```
┌──────────────────────────────────────────────────────────────────┐
│                         Observer                                 │
│                                                                  │
│  ┌──────────────┐    ┌───────────────────┐    ┌───────────────┐  │
│  │   Layer 1    │    │     Layer 2       │    │    Layer 3    │  │
│  │              │───▶│                   │───▶│               │  │
│  │ Raw Extractor│    │Semantic Interpreter│    │ State Tracker │  │
│  └──────────────┘    └───────────────────┘    └───────────────┘  │
│         │                     │                       │          │
│         ▼                     ▼                       ▼          │
│  RawPageSnapshot     SemanticPageModel        ObservationDiff    │
│                                                       │          │
└───────────────────────────────────────────────────────┼──────────┘
                                                        │
                                              ┌─────────▼─────────┐
                                              │     Adapter       │
                                              │ SemanticField  →  │
                                              │   FormField       │
                                              └─────────┬─────────┘
                                                        │
                                                        ▼
                                                 list[FormField]
                                                        │
                          ┌─────────────────────────────┼──────────┐
                          │                             │          │
                          ▼                             ▼          ▼
                    domhand_fill              fill_executor   verification
                    (unchanged)               (unchanged)     (unchanged)
```

**Data flow:**
1. Layer 1 collects raw facts from the browser → `RawPageSnapshot`
2. Layer 2 interprets those facts in Python → `SemanticPageModel`
3. Layer 3 diffs against previous observation → `ObservationDiff`
4. Adapter converts to `list[FormField]` for the existing action layer

## 4. Data Models

These are the new Pydantic models introduced by the rebuild. `FormField` and
`ExtractionResult` remain untouched — the adapter bridges between old and new.

### 4.1 RawElement (Layer 1 output)

A single element as collected from the browser. No interpretation — just facts.

```python
class RawElement(BaseModel):
    """A single DOM element with all raw signals collected from the browser.

    This is a fact record — no classification, no label resolution.
    The Semantic Interpreter consumes these to produce SemanticFields.
    """

    # ── Identity ──
    backend_node_id: int              # CDP BackendNodeId — stable across calls
    ff_id: str                        # data-ff-id tag (e.g. "ff-0") — set during extraction

    # ── DOM signals ──
    tag_name: str                     # "input", "div", "select", etc.
    input_type: str = ""              # HTML type attr: "text", "email", "tel", etc.
    attributes: dict[str, str] = {}   # All HTML attributes (data-*, aria-*, class, etc.)

    # ── ARIA signals (from DOM attributes) ──
    role: str = ""                    # role="combobox", role="radio", etc.
    aria_label: str = ""
    aria_labelledby: str = ""
    aria_required: bool = False
    aria_checked: str = ""            # "true", "false", "mixed"
    aria_expanded: str = ""           # "true", "false"
    aria_disabled: bool = False
    aria_haspopup: str = ""
    aria_controls: str = ""
    aria_owns: str = ""

    # ── A11y tree signals (from CDP Accessibility.getFullAXTree) ──
    ax_role: str = ""                 # Browser-computed role (the key new signal)
    ax_name: str = ""                 # Browser-computed accessible name
    ax_description: str = ""          # Browser-computed description
    ax_properties: dict[str, Any] = {}  # checked, expanded, required, disabled, etc.

    # ── Content ──
    text_content: str = ""            # Direct text content
    value: str = ""                   # Current value of the element
    placeholder: str = ""
    name_attr: str = ""               # HTML name attribute

    # ── Options (for selects, listboxes) ──
    options: list[dict[str, str]] = []  # [{value: "US", text: "United States"}, ...]

    # ── Spatial ──
    bounding_box: dict[str, float] | None = None   # {x, y, width, height}

    # ── Context ──
    is_visible: bool = True
    is_disabled: bool = False
    is_native_select: bool = False
    is_in_shadow_dom: bool = False
    parent_container_id: str = ""     # backend_node_id of nearest form/fieldset/section

    # ── Neighboring elements (for label/group resolution) ──
    preceding_text: str = ""          # Text content of prior sibling/element
    parent_text: str = ""             # Text content of parent container
    nearest_heading: str = ""         # Closest h1-h6/legend above this element
    label_for_text: str = ""          # Text of <label for="this.id">
```

### 4.3 SemanticField (Layer 2 output)

The core model of the new system. Represents one logical form field
after interpretation.

```python
class CanonicalType(str, Enum):
    """The 8 canonical field types. Every widget maps to one of these."""
    TEXT = "text"
    SELECT = "select"
    RADIO_GROUP = "radio_group"
    CHECKBOX_GROUP = "checkbox_group"
    DATE = "date"
    FILE = "file"
    TOGGLE = "toggle"
    BUTTON_GROUP = "button_group"


class LabelSource(str, Enum):
    """Where the label came from — ordered by reliability."""
    ARIA_LABELLEDBY = "aria_labelledby"       # Most reliable
    AX_COMPUTED_NAME = "ax_computed_name"      # Browser-computed
    ARIA_LABEL = "aria_label"
    LABEL_FOR = "label_for"
    LEGEND = "legend"
    SIBLING_HEADING = "sibling_heading"
    SIBLING_TEXT = "sibling_text"
    PLACEHOLDER = "placeholder"
    NAME_ATTR = "name_attr"
    SYNTHETIC = "synthetic"                    # Least reliable


class FieldConfidence(BaseModel):
    """How confident we are in our interpretation of this field."""
    type_confidence: float = 1.0       # 0.0 - 1.0
    label_confidence: float = 1.0      # 0.0 - 1.0
    type_signals: list[str] = []       # What signals informed the type decision
    label_source: LabelSource | None = None


class SemanticField(BaseModel):
    """A fully interpreted form field.

    This is the primary output of the Semantic Interpreter. Each SemanticField
    represents one logical field that the user needs to fill, with a canonical
    type, resolved label, and all metadata needed for filling.
    """

    # ── Identity ──
    field_id: str                     # Stable ID (ff_id from extraction)
    backend_node_id: int              # For correlation back to DOM/a11y
    fingerprint: str = ""             # Content-based fingerprint for tracking across extractions

    # ── Semantic interpretation ──
    canonical_type: CanonicalType     # One of 8 types — the key classification
    label: str                        # Resolved human-readable label
    section: str = ""                 # Which section this belongs to (e.g. "Work Experience")

    # ── State ──
    value: str = ""                   # Current value
    required: bool = False
    visible: bool = True
    disabled: bool = False

    # ── Options (for select, radio_group, checkbox_group, button_group) ──
    options: list[FieldOption] = []   # Reuses existing FieldOption model
    is_multi_select: bool = False
    is_native_select: bool = False

    # ── Group membership ──
    member_ids: list[str] = []        # For groups: IDs of individual member elements
    group_parent_id: str = ""         # For members: ID of the parent group

    # ── Date-specific ──
    component_field_ids: list[str] = []   # For segmented dates: IDs of sub-fields
    has_calendar_trigger: bool = False
    format_hint: str = ""

    # ── File-specific ──
    accept: str = ""                  # Accepted file types

    # ── Quality metadata ──
    confidence: FieldConfidence = FieldConfidence()
    warnings: list[str] = []          # e.g. ["low_label_confidence", "ax_role_missing"]

    # ── Raw reference ──
    raw_label: str = ""               # Original label text before normalization
    label_sources: list[str] = []     # All label sources that contributed
```

### 4.5 ObservationDiff (Layer 3 output)

```python
class FieldChange(BaseModel):
    """A single field-level change between two observations."""
    field_id: str
    change_type: str        # "added" | "removed" | "value_changed" | "state_changed"
    old_value: str = ""
    new_value: str = ""
    details: str = ""       # Human-readable description of the change


class ObservationDiff(BaseModel):
    """What changed between the previous and current observation.

    This is what gets sent to the LLM instead of a full page dump.
    Dramatically reduces context size between fill steps.
    """

    current: SemanticPageModel              # Full current state
    changes: list[FieldChange] = []         # What changed since last observation

    # ── Summary stats ──
    fields_added: int = 0
    fields_removed: int = 0
    values_changed: int = 0
    is_new_page: bool = False               # True if URL changed (no diff possible)

    # ── Completeness ──
    total_fields: int = 0
    filled_fields: int = 0
    required_empty: int = 0                 # Required fields with no value
```

## 5. Layer 1: Raw Extractor

**Responsibility:** Collect raw facts from the browser. No interpretation.

```
┌─────────────────────────────────────────────────┐
│                  RawExtractor                    │
│                                                  │
│  Inputs:                                         │
│    - Playwright Page                             │
│    - browser-use EnhancedDOMTreeNode (optional)  │
│                                                  │
│  Outputs:                                        │
│    - RawPageSnapshot                             │
│                                                  │
│  What it does:                                   │
│    1. Injects minimal JS to tag elements (ff-id) │
│    2. Queries DOM for interactive elements        │
│    3. Reads all attributes, text, values          │
│    4. Fetches a11y tree data via CDP (if avail)   │
│    5. Reads bounding boxes from snapshot          │
│    6. Reads neighboring text for label context    │
│    7. Returns everything as RawPageSnapshot       │
│                                                  │
│  What it does NOT do:                            │
│    ✗ Decide field types                          │
│    ✗ Resolve labels                              │
│    ✗ Group related elements                      │
│    ✗ Filter or skip elements                     │
│    ✗ Platform-specific logic                     │
└─────────────────────────────────────────────────┘
```

### Interface

```python
class ExtractionStrategy(Protocol):
    """Pluggable extraction backend — the Strategy pattern."""

    async def extract(self, page: Page) -> RawPageSnapshot:
        """Extract raw elements from the page."""
        ...


class DOMOnlyStrategy:
    """Strategy A: DOM-only extraction (closest to current system)."""

    async def extract(self, page: Page) -> RawPageSnapshot:
        # Uses page.evaluate() JS similar to current extractor
        # but JS only collects facts — no type resolution, no label climbing
        ...


class HybridDOMAXStrategy:
    """Strategy B: DOM + a11y tree extraction (target architecture)."""

    async def extract(self, page: Page) -> RawPageSnapshot:
        # Uses browser-use's DomService to get EnhancedDOMTreeNodes
        # which already contain merged DOM + a11y + snapshot data
        # Maps each node to a RawElement with both DOM and ax_ fields populated
        ...


class RawExtractor:
    """Layer 1 entry point."""

    def __init__(self, strategy: ExtractionStrategy):
        self._strategy = strategy

    async def extract(self, page: Page) -> RawPageSnapshot:
        return await self._strategy.extract(page)
```

**Why the Strategy pattern?** We can build `DOMOnlyStrategy` first (low risk,
mirrors current behavior) and then build `HybridDOMAXStrategy` (high value,
adds a11y tree) without changing Layer 2 or Layer 3. The two strategies can
run side-by-side during validation.

### 6.1 Type Classifier

Determines the canonical type of each element.

```python
class TypeClassifier:
    """Classifies a RawElement into one of 8 CanonicalTypes.

    Uses multi-signal scoring instead of first-match if/else.
    Returns the type AND a confidence score.
    """

    def classify(self, element: RawElement) -> tuple[CanonicalType, float, list[str]]:
        """Returns (type, confidence, signals_used)."""

        # Score each possible type based on all available signals
        scores: dict[CanonicalType, float] = {}
        signals: list[str] = []

        # ── Signal: a11y tree role (highest weight — browser-computed) ──
        # ax_role "combobox" → strong SELECT signal
        # ax_role "textbox"  → strong TEXT signal
        # ax_role "radio"    → strong RADIO_GROUP signal (individual, grouped later)
        # ax_role "checkbox"  → strong CHECKBOX_GROUP signal

        # ── Signal: HTML tag + type (medium weight) ──
        # <select>           → SELECT
        # <input type=text>  → TEXT
        # <textarea>         → TEXT
        # <input type=file>  → FILE

        # ── Signal: ARIA attributes (medium weight) ──
        # role="combobox"    → SELECT
        # aria-haspopup      → SELECT

        # ── Signal: Platform data attributes (low weight — fallback) ──
        # data-uxi-widget-type="selectinput" → SELECT

        # Pick the type with the highest score
        # Return confidence based on signal agreement
        ...
```

**Key difference from current system:** Instead of the if/else chain at
`field_extractor.py:129-155` that exits on the first match, this looks at
ALL signals and scores them. When signals agree (a11y says combobox AND
role says combobox), confidence is high. When they disagree, confidence drops
and a warning is emitted.

**Testability:** This is a pure function. You can test it with:
```python
def test_workday_select():
    raw = RawElement(
        tag_name="div",
        attributes={"data-uxi-widget-type": "selectinput"},
        aria_haspopup="listbox",
        ax_role="combobox",
    )
    type, confidence, signals = classifier.classify(raw)
    assert type == CanonicalType.SELECT
    assert confidence > 0.9
```

No browser needed. No Playwright. No page.evaluate(). Just a dictionary in, type out.

## 7. Layer 3: State Tracker

**Responsibility:** Track changes across observations. Produce diffs.

```
┌──────────────────────────────────────────────────────────┐
│                     StateTracker                          │
│                                                          │
│  Inputs:                                                 │
│    - Current SemanticPageModel                           │
│    - Previous ObservationState (from last step)          │
│    - MutationObserver dirty set (which elements changed) │
│                                                          │
│  Outputs:                                                │
│    - ObservationDiff (what changed)                      │
│    - Updated ObservationState (for next step)            │
│                                                          │
│  What it tracks:                                         │
│    - New fields appearing (conditional reveals)          │
│    - Fields disappearing (page sections collapsing)      │
│    - Value changes (confirming fills worked)             │
│    - State changes (checkbox checked, dropdown expanded) │
│    - Page transitions (URL changed)                      │
│                                                          │
│  MutationObserver usage:                                 │
│    - Injected JS watches for attribute/child changes     │
│    - Debounced — batches changes, not per-mutation       │
│    - Attribute-filtered — only watches value, checked,   │
│      aria-*, class changes                               │
│    - Reports a SET of changed element IDs                │
│    - Layer 3 only re-interprets changed elements         │
│    - NOT always-on — activated between fill steps only   │
└──────────────────────────────────────────────────────────┘
```

## 8. Backward Compatibility: The Adapter

**This is the most critical piece for safe migration.**

The entire action layer (`domhand_fill`, `fill_executor`, `verification_engine`,
20+ browser scripts) depends on `FormField` with exact field names and types.
We cannot change the consumer — we must produce compatible output.

```python
class FormFieldAdapter:
    """Converts SemanticField → FormField for backward compatibility.

    This adapter is the ONLY bridge between the new observation system
    and the existing action layer. It ensures the action layer never
    needs to change.

    The adapter will be removed when the action layer is eventually
    updated to consume SemanticField directly (future milestone).
    """

    # Mapping from new CanonicalType to old field_type strings
    TYPE_MAP = {
        CanonicalType.TEXT: "text",
        CanonicalType.SELECT: "select",
        CanonicalType.RADIO_GROUP: "radio-group",
        CanonicalType.CHECKBOX_GROUP: "checkbox-group",
        CanonicalType.DATE: "date",
        CanonicalType.FILE: "file",
        CanonicalType.TOGGLE: "toggle",
        CanonicalType.BUTTON_GROUP: "button-group",
    }

    def to_form_field(self, field: SemanticField) -> FormField:
        """Convert a SemanticField to a FormField.

        Must produce output identical to what the old extractor would
        have produced for the same element.
        """
        return FormField(
            ff_id=field.field_id,
            field_type=self.TYPE_MAP[field.canonical_type],
            label=field.label,
            raw_label=field.raw_label,
            value=field.value,
            required=field.required,
            visible=field.visible,
            disabled=field.disabled,
            options=field.options,
            choices=[opt.text for opt in field.options],
            section=field.section,
            is_native=field.is_native_select,
            is_multi_select=field.is_multi_select,
            accept=field.accept,
            label_sources=[field.confidence.label_source.value]
                          if field.confidence.label_source else [],
            observation_warnings=field.warnings,
            field_fingerprint=field.fingerprint,
            btn_ids=field.member_ids if field.canonical_type == CanonicalType.BUTTON_GROUP else [],
            component_field_ids=field.component_field_ids,
            has_calendar_trigger=field.has_calendar_trigger,
            format_hint=field.format_hint,
        )

    def to_extraction_result(self, model: SemanticPageModel) -> ExtractionResult:
        """Convert a full SemanticPageModel to an ExtractionResult."""
        return ExtractionResult(
            fields=[self.to_form_field(f) for f in model.fields],
            page_title=model.title,
            page_url=model.url,
            form_count=model.form_count,
            has_submit_button=model.has_submit_button,
        )
```

## 9. How the A11y Tree Fits In

The a11y tree is **not a separate system** — it's an additional data source
that flows through the same pipeline.

```
                    Current System
                    ═════════════

  Browser page
       │
       ▼
  DOM traversal (JS)  ──────────────────────▶  FormField
  (the only data source)



                    New System
                    ══════════

  Browser page
       │
       ├──▶  DOM traversal ──┐
       │                     │
       ├──▶  A11y tree (CDP) ├───▶  RawElement  ──▶  SemanticField
       │                     │      (has BOTH        (interpreted
       └──▶  Snapshot (CDP)  ┘       DOM and a11y     in Python)
                                     signals)
```

**The a11y tree provides three things the DOM cannot:**

| What | DOM gives you | A11y tree gives you |
|------|---------------|---------------------|
| **Widget type** | Tag name + attributes (platform-specific) | `ax_role`: browser-computed semantic role (generic) |
| **Label** | Manual climbing to find `<label>`, headings, siblings | `ax_name`: browser-computed accessible name (follows W3C spec) |
| **State** | Read attributes like `checked`, `aria-checked`, class names | `ax_properties.checked`: browser-computed actual state |

**Concrete example — "Is this field required?"**

Current system checks (in JS):
```javascript
el.required || el.getAttribute('aria-required') === 'true' || label.includes('*')
```

A11y tree gives:
```python
ax_properties = {"required": True}  # Browser already figured it out
```

The browser looked at `required`, `aria-required`, and any implicit requiredness
from the HTML spec and computed a single boolean. We just read the answer.

### When the a11y tree doesn't help

| Situation | Why | Fallback |
|-----------|-----|----------|
| Custom button groups (styled `<button>` elements) | No ARIA `radiogroup` role applied | DOM heuristic: check spatial proximity + shared container |
| Workday segmented dates | Three inputs, no ARIA grouping | DOM heuristic: check `component_field_ids` pattern |
| Oracle opaque select values | A11y shows selected text but not internal value | DOM heuristic: read `data-value` attributes |
| Cross-shadow-root `aria-labelledby` | Chrome doesn't resolve cross-root ID refs | DOM heuristic: manual label climbing |

These cases still need heuristics — but they're handled in the Semantic Interpreter
(Python, testable) not in the JS blob.

### 10.2 Where the new observer plugs in

The new observer replaces `extract_form_fields()` and
`extract_visible_form_fields()`. Everything downstream is untouched.

```python
# New top-level entry point (replaces extract_form_fields)

class Observer:
    """Top-level observation API. Drop-in replacement for extract_form_fields."""

    def __init__(
        self,
        strategy: ExtractionStrategy = None,  # Default: DOMOnlyStrategy
    ):
        self.extractor = RawExtractor(strategy or DOMOnlyStrategy())
        self.interpreter = SemanticInterpreter()
        self.state_tracker = StateTracker()
        self.adapter = FormFieldAdapter()

    async def observe(self, page: Page) -> ExtractionResult:
        """Full observation cycle. Returns ExtractionResult for backward compat."""

        # Layer 1: Extract raw facts
        snapshot = await self.extractor.extract(page)

        # Layer 2: Interpret semantically
        model = self.interpreter.interpret(snapshot)

        # Layer 3: Diff against previous state
        diff = self.state_tracker.diff(model)

        # Adapter: Convert to legacy format
        return self.adapter.to_extraction_result(diff.current)

    async def observe_semantic(self, page: Page) -> ObservationDiff:
        """New API returning rich observation with diffs.

        For future consumers that want SemanticField directly
        instead of FormField.
        """
        snapshot = await self.extractor.extract(page)
        model = self.interpreter.interpret(snapshot)
        return self.state_tracker.diff(model)
```

### Parallel shadow mode (Phase 4-5)

During validation, BOTH extractors run on every page:

```python
async def extract_with_shadow_validation(page):
    # Run old extractor
    old_result = await old_extract_form_fields(page)

    # Run new observer
    new_result = await new_observer.observe(page)

    # Compare outputs
    mismatches = compare_extraction_results(old_result, new_result)
    if mismatches:
        log.warning("observer.shadow_mismatch", mismatches=mismatches)

    # Return old result (safe) — new result is for logging only
    return old_result
```

This lets us validate the new observer against real production traffic
without any risk of breaking user-facing behavior.
